Language

Here, it is described generally how the language functions internally, referencing the internal structure of objects, inheritance and function-calling.

Objects

All values in hydra are objects. An object in hydra consists of several parts:

  1. Slots: The most important part of an object is it's slots, which are sometimes referred to as fields in other Object Oriented languages. A slot can be thought of as a (name, value) pair, which relates names - represented internally as objects called symbols to values (i.e. other objects). There is a third part of a slot which is optional, called a contract, which is part of inheritance in Hydra (more on this later).
  2. Meta: Objects also have a distinct set of slots called metadata-slots, which store (as the name implies) metadata. This information is harder to access and modify, and includes information that is useful to either the programmer (e.g. documentation) or the interpreter (e.g. debug information).
  3. Contract: Hydra is prototype-based, which means that objects can also play the role of /classes/ in other languages, forming the basis of inheritance hierarchies. A contract allows for more complex, parametric inheritance.
